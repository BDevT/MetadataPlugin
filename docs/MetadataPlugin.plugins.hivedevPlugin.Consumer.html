class Consumer(WorkerBase):

Constructor: Consumer(q_in)

View In Hierarchy

Worker thread that receives Markdown file paths from the producer and processes them. Consumer class is designed to continuously run in a thread, checking the input queue for data. When data is available in the queue, it emits a signaln (sigDataAvailable) with the retrieved data and proceeds with further tasks if any, continually checking the queue for more data to process.

def run(self): ¶

Main loop of the Consumer. Continuously checks the input queue for new paths. When a path is available, it retrieves it, emits a signal with the path, and marks the task as done in the queue. Stops when the stop event is set.
